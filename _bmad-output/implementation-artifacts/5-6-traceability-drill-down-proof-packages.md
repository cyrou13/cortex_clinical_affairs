# Story 5.6: Traceability Drill-Down & Proof Packages

Status: ready-for-dev

## Story

As an RA Manager or Auditor,
I want to click any CER claim and see the complete source chain in one click,
So that I can instantly justify any claim for audit purposes.

## Acceptance Criteria

**Given** a CER section with claims
**When** the user clicks on a claim
**Then** the TraceabilityDrillDown panel opens on the right showing the cascade:
Level 1: CER Claim (statement, section number)
Level 2: SOA Source (section, extracted data, benchmark)
Level 3: Validation Study (endpoint, result, comparison)
Level 4: SLS Article (title, DOI, query that found it) (FR52, FR82)
**And** each level is clickable to navigate to the source
**And** the system enforces 100% traceability -- CER cannot be finalized with unlinked claims (FR84, R6)
**And** auditors can export audit trails in CSV or PDF format (FR83)
**And** a "Export Proof Package" button generates a PDF with the complete claim chain + timestamps + audit trail (FR77)
**And** proof package export completes in <30 seconds

## Tasks / Subtasks

### Backend

- [ ] Create use case `apps/api/src/modules/cer/application/use-cases/get-claim-trace.ts`:
  - Accept claimTraceId or claimText + sectionId
  - Build complete traceability chain:
    - Level 1: CER Claim (text, section number, section title)
    - Level 2: SOA Source (section, extracted grid data, benchmark value)
    - Level 3: Validation Study (endpoint, result, SOA benchmark comparison)
    - Level 4: SLS Article (title, authors, DOI, query, database source)
  - Return structured trace data with all levels
  - Each level includes timestamps and user actions from audit trail
- [ ] Create use case `apps/api/src/modules/cer/application/use-cases/enforce-traceability.ts`:
  - Scan all CerSection content for claims/references
  - Cross-check against ClaimTrace records
  - Return traceability coverage percentage
  - Return list of untraced claims with section locations
  - Block CER finalization if coverage < 100% (throw `TraceabilityViolationError`)
- [ ] Create use case `apps/api/src/modules/cer/application/use-cases/export-proof-package.ts`:
  - Generate PDF proof package for a specific claim
  - Include: claim text, full trace chain (all 4 levels), audit trail entries, timestamps
  - Use DOCX generation engine (Carbone template) to create professional PDF
  - Enqueue via BullMQ queue `cer:export-proof-package`
  - Target completion: <30 seconds
- [ ] Create use case `apps/api/src/modules/cer/application/use-cases/export-audit-trail.ts`:
  - Accept cerVersionId and optional filters (dateRange, userId, actionType)
  - Query AuditLog entries related to the CER
  - Generate CSV or PDF format based on request
  - Include: WHO (user), WHAT (action), WHEN (timestamp), WHY (reason)
- [ ] Create proof package DOCX template at `apps/workers/src/shared/docx/templates/proof-package.docx`:
  - Header: CORTEX Proof Package, document title, generation date
  - Claim section: claim text, CER section reference
  - Trace chain: each level as a card with source details
  - Audit trail: chronological list of actions
  - Footer: checksum, generated by CORTEX
- [ ] Create BullMQ worker `apps/workers/src/processors/cer/export-proof-package.ts`:
  - Process proof package export job
  - Gather trace data + audit trail
  - Generate DOCX/PDF via Carbone template
  - Store generated file in MinIO
  - Return download URL
- [ ] Create `TraceChain` GraphQL type:
  - `level1`: CerClaimLevel (claimText, sectionNumber, sectionTitle)
  - `level2`: SoaSourceLevel (sectionTitle, extractedData, benchmark)
  - `level3`: ValidationLevel (studyName, endpoint, result, comparison)
  - `level4`: SlsArticleLevel (title, authors, doi, query, database)
  - `auditTrail`: [AuditEntry] (userId, action, timestamp)
- [ ] Create GraphQL queries:
  - `claimTrace(claimId: UUID!)` - get complete trace chain
  - `traceabilityCoverage(cerVersionId: UUID!)` - coverage metrics
  - `unresolvedClaims(cerVersionId: UUID!)` - list untraced claims
- [ ] Create GraphQL mutations:

## Dev Notes

### Technology Stack

- **Backend**: Fastify 5.7, Apollo Server 4, Prisma 7.2
- **DOCX/PDF Generation**: Carbone.io for proof package template, BullMQ for async export
- **Object Storage**: MinIO for generated proof packages
- **Frontend**: React 19, Apollo Client 3.x, shadcn/ui Sheet component

### Architecture Patterns

- **Cross-Module Query**: Trace resolution reads from SLS, SOA, Validation repositories (read-only, not write)
- **Traceability Enforcement**: Architecture-enforced - CER cannot be locked if traceability < 100% (R6)
- **Proof Package Export**: Async via BullMQ `cer:export-proof-package`, stored in MinIO, download URL returned
- **Audit Trail**: Read from shared AuditLog table, filtered by CER-related entries

### Traceability Chain Structure

```
Level 1: CER Claim
├── Claim text: "Sensitivity of CINA-CSpine: 93.9%"
├── Section: 8.2 - Clinical Data Analysis
└── Status: verified ✓

Level 2: SOA Source
├── SOA Device Section 3 - Performance Benchmarks
├── Benchmark: Sensitivity target >= 92%
└── Status: MET ✓

Level 3: Validation Study
├── Study: CINA-CSpine v1.0 Standalone
├── Endpoint: Sensitivity 93.9% (95% CI: 90.1-96.8%)
└── SOA Benchmark comparison: MET

Level 4: SLS Article
├── Article: Smith et al., 2024, "Cervical Spine Fracture Detection..."
├── DOI: 10.1234/example
├── Query: #Q3, Database: PubMed
└── Found: 2024-01-15
```

### UX Design Notes

- **TraceabilityDrillDown**: The "superpouvoir UX" of CORTEX per UX spec. Must be instant (< 5 seconds), intuitive, and satisfying.
- **Panel**: 380px right side, slide-in animation 200ms, uses Sheet shadcn base
- **Level Cards**: Connected by vertical dotted lines (blue-200), each with module-specific icon
- **Navigation**: Each level clickable to navigate to source. Click on Level 4 article opens article detail.
- **Export Button**: Always visible at top of drill-down panel, Primary style with download icon
- **Coverage Indicator**: Circular progress ring, green/orange/red based on percentage
- **Emotional Design**: "Confidence absolue" - auditor asks for justification, Marie exports proof package in 30 seconds
- **Accessibility**: `aria-label="Traceability navigation"`, Escape to close, focus management in panel
