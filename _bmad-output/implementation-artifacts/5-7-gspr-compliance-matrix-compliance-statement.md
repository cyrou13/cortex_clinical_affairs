# Story 5.7: GSPR Compliance Matrix & Compliance Statement

Status: ready-for-dev

## Story

As an RA Manager,
I want an auto-generated GSPR Compliance Matrix with compliance statement,
So that I can demonstrate compliance with General Safety and Performance Requirements.

## Acceptance Criteria

**Given** a CER with linked upstream modules
**When** the GSPR matrix is generated
**Then** the system auto-generates a GSPR Compliance Matrix with rows from applicable GSPRs and columns for evidence mapping (FR53)
**And** each GSPR row shows: requirement, status (compliant/partial/not applicable), evidence reference
**And** a GSPR compliance statement is auto-generated from the matrix statuses (FR58o)
**And** the compliance statement is included in the CER document (FR58p)
**And** the matrix is editable for manual adjustments
**And** the matrix is rendered in an ag-Grid with status color coding

## Dev Notes

### Technology Stack

- **Backend**: Fastify 5.7, Apollo Server 4, Prisma 7.2
- **Data Grid**: ag-Grid Enterprise 33 for GSPR matrix display
- **LLM**: LLM abstraction layer for compliance statement drafting
- **Frontend**: React 19, Apollo Client 3.x, ag-Grid, Plate editor

### Architecture Patterns

- **Auto-Generation**: GSPR matrix auto-populated from upstream data with manual override capability
- **Evidence Linking**: GsprEvidenceLink creates traceable connections between GSPRs and source data
- **LLM Drafting**: Compliance statement uses LLM to generate professional regulatory language
- **Editable Grid**: ag-Grid inline editing with backend persistence for status and notes changes

### GSPR Reference Structure

MDR Annex I is organized as:

- **Chapter I (GSPR 1-9)**: General Requirements - risk management, clinical evidence, performance characteristics
- **Chapter II (GSPR 10-22)**: Design and Manufacture - chemical, biological, infection, mechanical properties, software, devices with diagnostic function
- **Chapter III (GSPR 23)**: Information supplied with device - labeling, IFU

### UX Design Notes

- **ag-Grid Theme**: CORTEX tokens, header #F8F9FA, Inter 14px, status-colored accent bars
- **Color Coding**: Green (compliant), Orange (partial), Gray (not applicable) - never red (not an error)
- **Summary Header**: Large Stripe-style numbers above the grid
- **Inline Editing**: Click on status dropdown or notes field to edit inline (ag-Grid cell editing)
- **Evidence Linking**: Modal with upstream data browser, clean selection UI
- **Compliance Statement**: Plate editor for editing, regenerate button for AI re-draft
